### Importancia de la Versión de Java

La versión de Java utilizada para compilar y ejecutar una aplicación es crucial, ya que evita errores como el siguiente:

```
java.lang.UnsupportedClassVersionError: com/intellij/util/lang/PathClassLoader has been compiled by a more recent version of the Java Runtime (class file version 61.0), this version of the Java Runtime only recognizes class file versions up to 52.0
```

Este error indica que la aplicación ha sido compilada con una versión más reciente de Java de la que está instalada en tu sistema. Para evitar este error, necesitas asegurarte de que la versión de Java en tu sistema sea compatible con la versión utilizada para compilar la aplicación.

### Cambiar la Versión de Java en el Sistema

Para cambiar la versión de Java utilizada en tu sistema, puedes usar el comando `update-alternatives`, una herramienta que te permite seleccionar entre diferentes versiones de un mismo programa instalado.

<Steps>

1. **Verificar las versiones de Java instaladas:**

   Primero, verifica las versiones de Java disponibles en tu sistema con el siguiente comando:

   ```bash
   sudo update-alternatives --config java
   ```

   Esto te mostrará un menú con las versiones de Java instaladas, junto con el número que debes seleccionar para elegir una versión.

   <Aside type="tip">
   Si solo ves una opción, significa que solo tienes una versión de Java instalada.
   </Aside>

2. **Seleccionar la versión de Java:**

   En el menú, elige la versión de Java que deseas utilizar escribiendo el número correspondiente y presionando `Enter`.

   Por ejemplo, si el menú muestra las versiones `11` y `8`, y quieres cambiar a Java 8, selecciona el número correspondiente a Java 8.

   <Aside type="caution">
   Asegúrate de seleccionar una versión de Java que sea compatible con la aplicación que estás intentando ejecutar.
   </Aside>

3. **Verificar la versión activa de Java:**

   Para confirmar que has cambiado la versión de Java correctamente, ejecuta:

   ```bash
   java -version
   ```

   Esto mostrará la versión de Java que está actualmente activa en tu sistema.

</Steps>

### Configuración de Java para Proyectos Específicos

Si trabajas con un IDE como IntelliJ IDEA o Eclipse, es importante configurar el entorno de ejecución Java en el proyecto. Estos entornos permiten especificar la versión del JDK a usar sin afectar la versión global del sistema.

<Steps>

1. **Configurar la versión de Java en IntelliJ IDEA:**

   - Ve a `File > Project Structure`.
   - En la sección `Project`, selecciona el JDK que deseas usar.
   - Si el JDK que deseas no está listado, añade uno nuevo seleccionando la ruta del JDK.

2. **Configurar la versión de Java en Eclipse:**

   - Ve a `Window > Preferences > Java > Installed JREs`.
   - Selecciona la versión de Java que deseas usar o añade una nueva ruta de JDK si es necesario.

</Steps>

### Solución de Problemas Comunes

1. **Problemas de bytecode incompatible:**

   Si recibes errores relacionados con `UnsupportedClassVersionError`, es probable que estés intentando ejecutar código que ha sido compilado con una versión más reciente de Java de la que está instalada en tu sistema. Asegúrate de que tu entorno de ejecución y de compilación estén alineados.

2. **Java no cambia después de usar `update-alternatives`:**

   Si después de cambiar la versión de Java con `update-alternatives`, el cambio no se refleja, intenta cerrar sesión y volver a iniciarla, o abre una nueva terminal.
